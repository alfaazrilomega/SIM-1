## Brief overview
This rule ensures that I implement comprehensive accessibility and inclusive design standards for Awwwards-level websites, ensuring they are usable by everyone regardless of abilities or disabilities.

## WCAG 2.1 AA Compliance Standards
- **Perceivable**: Ensure all content is presented in ways that users can perceive
- **Operable**: Make all functionality available through keyboard navigation
- **Understandable**: Make content readable and predictable
- **Robust**: Ensure compatibility with assistive technologies

## Screen Reader Optimization
- **Semantic HTML**: Use proper HTML5 semantic elements (header, nav, main, section, article, footer)
- **ARIA Labels**: Implement appropriate ARIA labels and roles for complex components
- **Alt Text**: Provide descriptive alt text for all images and visual content
- **Form Labels**: Ensure all form inputs have proper labels and instructions

## Keyboard Navigation Standards
- **Tab Order**: Implement logical tab navigation order
- **Focus Indicators**: Ensure visible focus indicators for all interactive elements
- **Skip Links**: Provide skip navigation links for long pages
- **Modal Accessibility**: Ensure modals trap focus and are dismissible via keyboard

## Visual Accessibility Features
- **Color Contrast**: Maintain minimum 4.5:1 contrast ratio for text, 3:1 for large text
- **Text Resizing**: Support text resizing up to 200% without loss of functionality
- **Color Independence**: Ensure information is not conveyed by color alone
- **Motion Sensitivity**: Provide options to reduce or disable animations for motion-sensitive users

## Cognitive Accessibility
- **Clear Language**: Use simple, clear language and avoid jargon
- **Consistent Navigation**: Maintain consistent navigation and layout patterns
- **Error Prevention**: Provide clear error messages and prevention strategies
- **Loading States**: Implement clear loading and progress indicators

## Assistive Technology Compatibility
- **Screen Reader Testing**: Test with popular screen readers (NVDA, JAWS, VoiceOver)
- **Screen Magnifier Support**: Ensure compatibility with screen magnification tools
- **Voice Control**: Support voice control navigation where applicable
- **Switch Access**: Ensure compatibility with switch access devices

## Mobile Accessibility
- **Touch Targets**: Ensure minimum 44px touch target sizes
- **Gesture Alternatives**: Provide alternatives to complex gestures
- **Orientation Support**: Support both portrait and landscape orientations
- **Mobile Screen Readers**: Test with mobile screen readers (TalkBack, VoiceOver)

## Inclusive Design Principles
- **Universal Design**: Design for the widest possible range of users
- **Cultural Sensitivity**: Consider cultural differences in design and content
- **Language Support**: Support multiple languages and right-to-left text
- **Age Inclusivity**: Design for users of all ages, including older adults

## Accessibility Testing & Validation
- **Automated Testing**: Use tools like axe, Lighthouse, and WAVE for automated checks
- **Manual Testing**: Conduct manual keyboard navigation and screen reader testing
- **User Testing**: Include users with disabilities in testing processes
- **Regular Audits**: Perform regular accessibility audits and updates

## Awwwards-Level Accessibility Standards
- **Beyond Compliance**: Exceed basic WCAG requirements for premium accessibility
- **Innovative Solutions**: Implement cutting-edge accessibility features
- **Performance & Accessibility**: Ensure accessibility features don't compromise performance
- **Accessibility Documentation**: Provide comprehensive accessibility documentation
