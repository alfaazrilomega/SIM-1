## Brief overview
This rule ensures that I manage the complete project lifecycle with proper timeline management, milestone tracking, and workflow optimization for Awwwards-level full-stack website development.

## Project Planning & Timeline Management
- **Project Scope Definition**: Clearly define project scope, deliverables, and success criteria
- **Timeline Planning**: Create realistic timelines with milestones for design, development, testing, and deployment phases
- **Resource Allocation**: Optimize resource allocation based on project complexity and requirements
- **Risk Assessment**: Identify potential risks and create mitigation strategies

## Milestone Tracking & Progress Monitoring
- **Phase Breakdown**: Divide project into clear phases (research, design, frontend, backend, integration, testing, deployment)
- **Milestone Validation**: Validate completion of each milestone before proceeding to the next phase
- **Progress Reporting**: Provide regular progress updates with clear status indicators
- **Quality Gates**: Implement quality gates at each milestone to ensure standards are met

## Workflow Optimization
- **Agile Methodology**: Apply agile principles with iterative development and continuous feedback
- **Task Prioritization**: Prioritize tasks based on impact, dependencies, and user value
- **Parallel Development**: Identify opportunities for parallel development to optimize timeline
- **Dependency Management**: Track and manage dependencies between frontend, backend, and design components

## Project Communication & Collaboration
- **Regular Check-ins**: Schedule regular check-ins to review progress and address issues
- **Issue Resolution**: Implement systematic approach to identify, track, and resolve project issues
- **Stakeholder Communication**: Maintain clear communication with all stakeholders throughout the project
- **Documentation Standards**: Ensure comprehensive documentation for all project phases

## Quality Assurance & Delivery
- **Definition of Done**: Establish clear criteria for when each component is considered complete
- **Integration Testing**: Plan comprehensive integration testing between all components
- **Performance Validation**: Validate performance benchmarks at each major milestone
- **Final Review Process**: Implement thorough final review before project delivery

## Project Success Metrics
- **Timeline Adherence**: Track adherence to planned timeline and milestones
- **Quality Standards**: Measure against Awwwards-level quality benchmarks
- **User Experience**: Validate user experience meets project goals
- **Technical Excellence**: Ensure technical implementation follows best practices
